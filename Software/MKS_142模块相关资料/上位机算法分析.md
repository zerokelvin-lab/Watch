# MKS-142 上位机算法分析

## 0. 功能概述

MKS-142上位机是一款基于.NET Framework 4.8开发的健康监测数据可视化软件，配合MKS-142健康检测模块使用。主要功能如下：

### 核心功能
- **串口通信**: 通过串口接收MKS-142模块发送的88字节健康数据帧
- **实时波形显示**: 以曲线图形式实时绘制PPG脉搏波、呼吸波等波形数据
- **健康指标展示**: 显示心率、血氧、微循环、疲劳指数、收缩压/舒张压、呼吸率、RR间期等指标
- **HRV分析图表**: 展示RR间期趋势图、血氧状态图
- **异常报警**: 心率或血氧异常时弹出提示

### 数据管理
- **数据记录**: 将采集数据保存为CSV文件（含时间戳、各项指标）
- **历史查询**: 查看历史测量记录
- **数据上传**: 通过HTTP POST将数据上传至景帆科技云平台（hapi.jingfantech.com）

### 系统功能
- **用户登录**: 支持账号密码登录云端平台
- **密码修改**: 支持修改登录密码
- **串口配置**: 自动检测可用串口，支持手动选择
- **设备绑定**: 通过channel_id和channel_secret绑定设备

### 技术特点
- .NET Framework 4.8 + WinForms
- 使用System.IO.Ports进行串口通信
- 使用Newtonsoft.Json进行API数据序列化
- 使用DES加密保护用户密码
- 使用Windows多媒体定时器实现毫秒级波形刷新

## 1. 通信协议

### 1.1 串口配置
- 波特率: 从配置文件读取
- 数据位: 8
- 停止位: 1
- 校验位: 无

### 1.2 数据帧格式 (88字节)
```
字节偏移    长度    字段名          说明
────────────────────────────────────────────────────
[0]         1       Header          帧头标识 (0xFF)
[1..64]     64      AdcData         波形数据 (PPG原始数据)
[65]        1       HeartRate       心率 (bpm)
[66]        1       SpO2            血氧饱和度 (%)
[67]        1       Bk              微循环指数
[68]        1       Dc              疲劳指数
[69]        1       Bbt             (预留/体温相关)
[70]        1       (未使用)
[71]        1       Sbp             收缩压 (mmHg)
[72]        1       Dbp             舒张压 (mmHg)
[73]        1       rr              呼吸率 (次/分)
[74]        1       rp              呼吸深度
[75]        1       RRInterval      RR间期
[76]        1       (未使用)
[77]        1       rmsdd           RMSSD (ms)
[78..79]    2       nn50/pnn50      NN50/PNN50
[80..83]    4       rra             RR间期数组
[84..87]    4       rsv             预留
```

## 2. 协议解析流程

### 2.1 serialPort1_DataReceived (串口接收)
```
1. 读取串口数据到缓冲区
2. 遍历每个字节:
   - 如果字节 == 0xFF 且 headflag == false:
     * 设置 headflag = true
     * jfPackagebufIndex = 0
     * 启动新线程处理
   - 如果 headflag == true:
     * 将字节存入 receivedPackageData[jfPackagebufIndex]
     * jfPackagebufIndex++
     * 如果 jfPackagebufIndex >= 519 (0x0207):
       - 调用 MatchProtocol()
       - jfPackagebufIndex = 0
```

### 2.2 MatchProtocol (协议匹配)
```
1. 调用 SerialPortClass.versionFiltration(data) 进行版本过滤
2. 调用 SerialPortClass.isHave255(receivedPackageData) 检查数据有效性
3. 如果 headflag == true:
   - 调用 AnalysisProtocol(data) 解析数据
4. 检查 adcDLen (ADC数据长度):
   - 如果 adcDLen == 特定阈值:
     * 设置 isreport = true
     * 重置 adcDLen = 0
     * 调用 ShowReport() 生成报告
   - 否则:
     * 重置 isreport = false
     * adcDLen++
```

## 3. 数据结构

### 3.1 SSRecData (传感器数据)
```c
struct SSRecData {
    uint8_t Header[1];      // 帧头
    uint8_t AdcData[64];    // PPG波形数据
    uint8_t HeartRate[1];   // 心率
    uint8_t SpO2[1];        // 血氧
    uint8_t Bk[1];          // 微循环
    uint8_t Dc[1];          // 疲劳指数
    uint8_t Bbt[2];         // 体温
    uint8_t Dbp[1];         // 舒张压
    uint8_t Sbp[1];         // 收缩压
    uint8_t rr[1];          // 呼吸率
    uint8_t rp[1];          // 呼吸深度
    uint8_t RRInterval[1];  // RR间期
    uint8_t sdann[1];       // SDANN
    uint8_t rmsdd[1];       // RMSSD
    uint8_t nn50[6];        // NN50
    uint8_t pnn50[2];       // pNN50
    uint8_t rra[2];         // RR间期数组
    uint8_t rsv[4];         // 预留
};
```

### 3.2 TJRecData (体检数据包)
```c
struct TJRecData {
    uint8_t header[2];      // 包头
    uint8_t sequence[2];    // 序列号
    uint8_t id[4];          // 设备ID
    uint8_t data[168];      // 数据 (168 = 0xA8)
    uint8_t rsv1[6];        // 预留
    uint8_t CRC16[2];       // CRC16校验
};
// 总长度: 196 字节 (0xC4)
```

## 4. 波形显示算法 (TimerShowWave)

```c
void TimerShowWave() {
    // 1. 检查 SpO2 数据有效性
    if (ssrecData.SpO2[0] == null) return;
    
    // 2. 更新主波形 (5通道)
    // 通道: SpO2[0], rr[0], HeartRate[0], Bk[0], Dc[0]
    curveControl1.DataBindY(
        ssrecData.SpO2[0],
        ssrecData.rr[0], 
        ssrecData.HeartRate[0],
        ssrecData.Bk[0],
        ssrecData.Dc[0]
    );
    
    // 3. 更新副波形
    curveControl1.DataBindY(
        ssrecData.SpO2[0],
        ssrecData.rr[0],
        ssrecData.HeartRate[0],
        ssrecData.Bk[0],
        ssrecData.Dc[0]
    );
    
    // 4. 更新RR间期图
    if (isreport) {
        showRRNew1.ShowRRNewImage(
            ssrecData.nn50[0],
            ssrecData.nn50[1],
            ssrecData.nn50[2],
            ssrecData.nn50[3],
            ssrecData.nn50[4]
        );
    }
    
    // 5. 更新血氧状态图
    if (isreport) {
        bloodstates1.ShowBloodstate(
            ssrecData.Sbp[0],
            ssrecData.Dbp[0],
            ssrecData.rr[0],
            ssrecData.rp[0],
            ssrecData.rr[0]
        );
    }
    
    // 6. 更新ADC波形数据
    for (int i = 0; i < 6; i++) {
        if (ssrecData.AdcData[i] != null) {
            // 绘制波形点
            curveControl1.AddPoint(ssrecData.AdcData[i]);
            adcDLen++;
        }
    }
    
    // 7. 检查是否需要生成报告
    if (adcDLen > threshold) {
        // 触发报告生成
    }
}
```

## 5. 数据上传 (SaveTJPack)

```c
void SaveTJPack(byte[] prot, int count) {
    // 初始化 TJRecData 各字段
    tjrecData.header = new byte[2];     // prot[0..1]
    tjrecData.sequence = new byte[2];   // prot[2..3]
    tjrecData.id = new byte[16];        // prot[4..19]
    tjrecData.data = new byte[168];     // prot[20..187] (0xA8)
    tjrecData.rsv1 = new byte[6];       // prot[188..193]
    tjrecData.CRC16 = new byte[2];      // prot[194..195]
    
    // 复制数据
    Buffer.BlockCopy(prot, 0, tjrecData.header, 0, 2);
    Buffer.BlockCopy(prot, 2, tjrecData.sequence, 0, 2);
    Buffer.BlockCopy(prot, 4, tjrecData.id, 0, 16);
    Buffer.BlockCopy(prot, 20, tjrecData.data, 0, 168);
    Buffer.BlockCopy(prot, 188, tjrecData.rsv1, 0, 6);
    Buffer.BlockCopy(prot, 194, tjrecData.CRC16, 0, 2);
    
    // 启动上传线程
    Thread t = new Thread(packsavepostThread);
    t.Start();
}
```

## 7. 嵌入式移植建议

### 7.1 需要移植的核心功能
1. **串口数据接收** - 88字节帧解析
2. **数据结构定义** - SSRecData
3. **波形数据处理** - 64字节ADC数据
4. **健康指标计算** - 心率、血氧、血压等
5. **HRV分析** - RR间期、SDNN、RMSSD、NN50、PNN50

### 7.2 可选功能
1. **数据存储** - CSV格式保存
2. **数据显示** - LCD波形绘制
3. **数据上传** - HTTP POST (需要网络模块)

### 7.3 STM32 资源需求
- RAM: 至少 1KB (数据缓冲)
- Flash: 至少 32KB (算法代码)
- UART: 1个 (传感器通信)
- 定时器: 1个 (波形刷新)
